挖掘文件系统往往并非乐趣所在，大多数人更倾向于安守 /home/ 目录。然而，适当探索系统提供的其他功能并尝试在新语境下思考它们，却能带来意想不到的收益。本文重点推荐了一个常被忽视的目录——/tmp/，它专为临时文件设计，会在系统重启时自动清空。作者通过将其作为下载目录、临时工作区及草稿本，有效解决了文件堆积和空间浪费的问题。通过利用 /tmp/ 的临时特性，用户可以更清晰地界定哪些数据需要长期保存，哪些可以安全丢弃，从而构建一个自我整理的数字工作空间。

你应该更多地使用 /tmp/

挖掘文件系统可能并非许多人的乐趣所在，大多数人满足于生活在 /home/ 目录中，仅此而已。但是，进行一些探索，看看你的机器提供了哪些其他功能，并尝试在新的语境下思考它们，可能会有好处。

我认为真正应该得到更多关注、且我不断发现新用例的目录是 ZZZTOKENPLACEHOLDER0ZZZ。/tmp/ 专为临时文件设计，通常是程序长期不需要的“东西”的归宿：你正在处理文件的临时备份、浏览器缓存的内容、正在进行的更新暂存区等。/tmp/ 还有一个独特的好处：每当你的机器重置时，它都会被自动清除——毕竟它是临时的。

/tmp/ 主要是作为软件执行诸如上述操作的空间，但没有任何限制阻止你将其用于自己的目的。因此，在过去的几年里，我确实将 /tmp/ 深度集成到了常规使用中，并取得了令人惊讶的效果。

将下载目录设为 /tmp/

我发现最强大的用途之一是将下载文件夹设置为 /tmp/。根据我的经验以及观察他人的做法，默认的“下载”文件夹似乎就像一个永远不会被清空的垃圾箱。千兆字节的数据就静静地躺在一个完全被遗忘的文件夹中：为什么我仍然保留着十多年前的 Ubuntu 12.04 ISO 文件？为什么几年前完成的项目还留在这里？尽管我使用 Linux 已有十多年，但那些 .exe 文件似乎依然存在。

挖掘自己的下载文件夹就像在翻阅一个令人着迷的时间胶囊，但你不必像进行考古挖掘一样偶然撞见这些文件。/tmp/ 是发送下载的绝佳位置。

  • 打印包装标签？将其发送到 /tmp/，打印出来，然后将其遗忘，而不是让它滞留在下载文件夹中。
  • 快速项目素材？有想要在 GIMP 中快速项目使用的图像吗？将其放入 /tmp/，而不是几年后还在疑惑为什么系统里会有一个看似随机的 JPEG 文件。

提升“严肃”工作的效率

利用 /tmp/ 作为我一般工作流程的一部分，对于更“严肃”的工作也出奇地有效。

特别是在进行研究时，我可能需要翻阅数百份 PDF 格式的研究论文。我可以将所有这些文件下载到我的 /tmp/ 文件夹，然后直接开始处理。它们不存在被丢弃在文件夹中并被遗忘的风险，也不存在完成后继续占用空间的风险。

为项目处理数据集？将其放入 /tmp/。当你回来继续工作时，只需下载最新版本即可。没有稳定地积累 v1、v2、v2_final 等版本文件。

完美的草稿本与 TODO 清单

快速 TODO 和其他类似草稿本的内容也非常适合 /tmp/。我过去常常只是在我的主目录中创建注释文件，例如 monday.md 或 2023-03-05.txt，记录当天需要完成的事项。/tmp/ 继续非常适合这种用途。我可以从在 /tmp/ 中创建一个文本文件开始新的一天，填入当天需要完成的工作，然后在关闭机器时将其自动清除。

优化 LaTeX 编译流程

我经常使用 LaTeX，它在编译文档时会生成大量文件：目录文件、参考文献文件、生成 PDF 过程中的中间 DVI 文件、生成的 PDF 等。这些文件确实有用，保持它们随时可用可以加快未来的编译速度，但更多时候我真的不需要它们。

因此，我可以在 pdflatex 命令中添加 -output-directory=/tmp/ 参数，将所有临时文件归位：

pdflatex -output-directory=/tmp/

这确实也会将生成的 PDF 放入 /tmp/，但通常我只希望保留源代码，并仅在需要时重新编译文档，从而节省空间。

构建自我整理的工作空间

虽然我在这里建议的任何想法都可以在组织良好的常规文件系统中实现，但在 /tmp/ 中工作时，很难忽视那种“所有未保存的内容都将丢失”的温和推动。

大量使用 /tmp/ 迫使你围绕机器上真正重要的内容做出决定：是保留机器上发生过的所有事情的副本，还是将真正需要的内容长期存储，而将其他所有内容视为一次性消耗品？这是一个自我整理的工作空间，没有随机文件最终闲置在系统中的烦恼。

/tmp/ 提供了许多令人惊讶的应用，这些应用纯粹源于它是临时的这一事实。

  • 它通过自动删除不需要的文件来帮助监控存储使用情况。
  • 它通过固有地清理不需要的内容来保持整个文件系统的整洁。

因此，请考虑尝试使用 /tmp/。如果它不适合你，你只需在下次重启后重新开始即可。

译者总结

/tmp/ 目录不仅是系统清理的垃圾场，更是构建高效、整洁个人工作流的利器。通过利用其自动清空的特性，我们可以强制自己区分“临时数据”与“长期资产”，从而避免数字囤积，让存储空间始终服务于当前的核心任务。
